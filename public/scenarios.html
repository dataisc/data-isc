Deep dives on selected simulator scenarios — assumptions, mechanics, and interpretation

The scenario deep-dives below relate exclusively to the built-in GDP Trajectories demo dataset. When you import your own data and define what-if scenarios, the engine will apply the same year-by-year projection logic to your dataset — scenario explanations specific to your data will be generated based on your inputs.

The GDP demo includes scenarios covering structural policy choices, trade dynamics, and environmental transitions — each applying growth rate adjustments to affected countries over a defined window. This section documents selected scenarios in depth.

How scenarios work

All scenarios apply growth rate deltas to affected countries over a defined window. Some represent sustained structural changes; others phase out gradually as the economy adjusts. See Methodology for the mechanics.

Running multiple scenarios

The simulator supports stacking multiple scenarios simultaneously. Deltas are additive — there is no interaction term. This means stacking a +1pp trade liberalisation with a −0.5pp carbon tax produces a net +0.5pp effect for countries affected by both. Non-linear interactions are not modelled.

Comparing two branches (GitDiff)

The Trajectories chart includes a ⑂ Branch mode for comparing two futures head-to-head. Pick a focus country and a branch year, then assign each branch its own stack of scenarios — for example Branch A: high automation versus Branch B: severe demographic strain. Both timelines are computed independently and drawn on the same graph, splitting from the branch year onward. A shaded band fills the gap between them, and delta cards report the divergence by the final year and the year the gap is widest. It is a fast way to see not just what each scenario does, but how two rival assumptions pull apart over time.

Proposing new scenarios

Scenario definitions live in scenarios_policy.json in the repository. If you want to propose a new scenario, open a pull request with the scenario definition and a short explanation of the effect parameters. See the GitHub repository for the schema.
