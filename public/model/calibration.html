Calibration

How growth rates are anchored to historical data and long-run consensus

This page describes the built-in demo dataset only — GDP Trajectories. It does not apply to datasets you import.

Approach

The calibration procedure fits nominal growth rate paths to minimise deviation between modelled GDP and the 1990–2025 IMF historical series, subject to convergence constraints. The fitted paths are then extrapolated to 2100 using the conditional convergence equations described in Methodology.

Terminal growth rates by income group

Terminal rates are nominal (real + inflation/nominal USD component) and represent the long-run attractor each economy converges toward.

Income GroupTerminal Nominal RateConvergence Speed (λ)Examples
Advanced3.5–4.5%0.025USA, Germany, Japan, UK
Emerging5.0–7.0%0.035China, Brazil, Turkey, Mexico
Developing6.5–9.5%0.045India, Nigeria, Bangladesh, Ethiopia

Country-level overrides

Twelve countries with structurally unusual growth patterns (commodity-dominated economies, city-states, post-conflict rebuilders) receive individual calibration overrides rather than group parameters. These include Singapore, Qatar, Norway, UAE, Libya, and Venezuela. The override values are stored server-side.

Nominal vs real calibration

The model calibrates to nominal USD GDP because that is the primary input data. The implicit nominal rate for any country bundles:

  • Real GDP growth
  • Domestic inflation relative to US inflation
  • Exchange rate movement against USD

These components are not separated. This means the model inherits the empirical complexity of historical nominal exchange rate dynamics without needing to model each component explicitly.

Validation

The calibration was validated by holding out 2015–2025 data and comparing model projections from a 2014 start to IMF actuals. Mean absolute percentage error (MAPE) across the 100-country set was 4.2% at the 5-year horizon and 11.8% at the 10-year horizon. This is comparable to published long-range IMF WEO forecast errors.

Updating calibration

Calibration is updated annually following the April IMF WEO release. The calibration script is included in the repository (model/calibrate.js) so that contributors can reproduce and propose improvements. See the GitHub repository for instructions.
