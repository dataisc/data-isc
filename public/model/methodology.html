Methodology

How the engine projects GDP from 2025 to 2100

This page describes the built-in demo dataset only — GDP Trajectories. It does not apply to datasets you import.

Core projection equation

Each country's GDP is projected one year at a time using a compounding nominal growth rate:

GDP(t+1) = GDP(t) × (1 + g_nominal(t))

Where g_nominal(t) is the country's nominal growth rate for year t, derived from a calibrated baseline rate adjusted for maturity, convergence, and any active scenario shocks.

Nominal vs real

All projections are nominal USD. The nominal growth rate implicitly bundles real GDP growth and a USD nominal inflation/exchange rate component. No deflator is applied. This is a design choice: comparing countries across time in nominal USD is the most commonly used frame for global economic scale comparisons, and introducing a deflator would require additional assumptions that themselves carry significant uncertainty at a 75-year horizon.

Conditional convergence

The model implements a soft form of conditional convergence: lower-income economies have higher baseline growth rates that decay as per-capita income rises. Advanced economies' rates converge toward a long-run terminal rate. This is consistent with the empirical growth literature (Barro & Sala-i-Martin, 1992; subsequent literature).

g_baseline(t) = g_terminal + (g_initial - g_terminal) × exp(-λ × (t - 2025))

Where λ is the convergence speed parameter, calibrated per income group. See Calibration for parameter values by group.

Scenario shock mechanics

When a scenario is applied, the engine adds a delta to the affected countries' growth rates over the scenario window:

g_scenario(t) = g_baseline(t) + δ(t)

The shock function δ(t) ramps up, sustains, and then decays back toward zero over the scenario's duration_years and recovery_years. The shape is:

  • Ramp-in: linear over the first 20% of duration
  • Sustained: constant at peak delta
  • Recovery: exponential decay toward zero over recovery_years
The severity field on each scenario card maps to a peak delta range. Mild = ±0.3–0.8 pp, Moderate = ±0.8–2.0 pp, Severe = ±2.0–5.0 pp. Exact values are server-side only and not exposed to clients.

Multi-scenario stacking

When multiple scenarios are active simultaneously, their deltas are summed before application. There is no interaction term — shocks are additive. This is a simplification; in reality, policy interactions can be non-linear.

Scope resolution

Each scenario specifies a default scope (global, regional, or country-specific). Users can override this. The scope determines which countries receive the shock delta; all others remain at baseline.

What is not modelled

  • Inflation and price level dynamics
  • Exchange rate movements (all projections assume a stable USD anchor)
  • Demographic transitions and labour force participation
  • Capital accumulation and investment rates
  • Technological discontinuities (AGI, energy transitions)
  • Financial crises, sovereign defaults, and non-linear tipping points
  • Climate physical risk (separate from carbon tax/green transition scenarios)

What-if scenarios for imported datasets

How the scenario engine works when you upload your own data

Core adjustment formula

For each entity and each year within the scenario window, the engine applies a compound growth adjustment on top of the baseline value:

adjusted_value(t) = baseline_value(t) × (1 + δ/100)(t − t₀ + 1)

Where δ is the annual impact rate (%) and t₀ is the scenario start year. The exponent starts at 1 so the effect is visible immediately at t₀ — a +2%/yr scenario shows +2% in the first year, +4.04% in the second, and so on. Values are clamped to non-negative.

Impact priority chain

When multiple sources specify a rate for the same entity and year, the engine applies the highest-priority source and ignores the rest:

PrioritySourceWhen active
1AI per-entity impactsAI provider connected; impacts reviewed and confirmed in the review modal
2Per-entity overridesEntity is checked in the Per-Entity tab with its own δ%/yr
3Phase rateCurrent year falls inside a defined phase window
4Global default δFallback for all in-scope entities

Scenario builder

Click + Add / edit in the What-If section, then + Add scenario. Each scenario card has four tabs:

  • Basic — name, description, year range, global default δ%/yr, optional value ceiling
  • Per-Entity — entity-specific δ%/yr; unchecked entities are excluded from the scenario
  • Phases — time-segmented rates (e.g. aggressive growth 2010–2015, then slowdown 2015–2020)
  • ↑ JSON — upload or paste a scenario JSON file; download a pre-filled template with your dataset's entity names

AI scenario generation (imported datasets)

When an AI provider is connected, the Describe a scenario with AI panel appears above the scenario cards. Click Generate & review and describe the scenario in plain language. The AI returns per-entity impact estimates; a review modal opens before anything is applied:

  • Every affected entity is listed with its AI-generated Δ%/yr, year window (From / To), and a one-line rationale
  • All values are editable inline — adjust any entity's rate or time window
  • Uncheck a row to exclude that entity from the scenario entirely
  • Rename the scenario before confirming
  • Click Create scenario → to confirm; click Discard to cancel without creating anything

Once created, the scenario card's Per-Entity tab is pre-populated with the reviewed values and can be edited further at any time using the ✎ edit button on the scenario pill.

Custom scenario JSON format

Scenarios can be written as standalone JSON files and shared or version-controlled independently of the application. Download a template from the ↑ JSON tab — it will be pre-filled with your dataset's entity names.

{
  "id": "green_transition_2025",
  "name": "Green Transition",
  "description": "Accelerated shift to renewables — asymmetric impact by energy mix.",
  "version": "1.0",
  "author": "your-github-handle",
  "default_delta_pct": 1.5,
  "year_from": 2025,
  "year_to": 2040,
  "growth_ceiling": null,
  "entity_overrides": [
    { "entity": "Norway",  "delta_pct": 3.2 },
    { "entity": "Germany", "delta_pct": 2.1 },
    { "entity": "Poland",  "delta_pct": -0.8 }
  ],
  "phases": [
    { "year_from": 2025, "year_to": 2030, "delta_pct": 2.5 },
    { "year_from": 2030, "year_to": 2040, "delta_pct": 1.0 }
  ]
}
FieldTypeDescription
default_delta_pctnumberGlobal annual adjustment (%). Applied to all entities not in entity_overrides.
year_from / year_tonumberScenario window (inclusive on both ends).
entity_overridesarrayPer-entity rates. delta_pct: null inherits default_delta_pct. Omitting an entity excludes it from the scenario.
phasesarrayTime windows with their own rate. Each phase overrides default_delta_pct for its year range.
growth_ceilingnumber | nullCap every adjusted value at this maximum — useful for percentage datasets (e.g. 100). null = no cap.

Difference from the GDP built-in model

The GDP demo uses a structural simulation: year-by-year growth rates are modulated by debt levels, demographic aging tiers, fossil resource depletion, and a multilateral cooperation index. Scenario effects are further differentiated by country attributes (trade openness, tech readiness, fossil dependency) via the countryModifier() function in compute.js.

For imported datasets, the engine applies the compound formula above. Without AI or per-entity overrides, all in-scope entities receive the same δ — this is a sensitivity analysis, not a structural simulation. Use per-entity overrides or AI-assisted analysis to encode asymmetric, domain-specific effects comparable to the GDP model's country attributes.

Contributing community scenarios

Scenario JSON files are self-contained and designed to be shared. To contribute to the main codebase: author your JSON, test it against one of the samples/ datasets, and open a pull request adding the file under samples/scenarios/. See the GitHub repository for the contributing guide.
